先确定你的部署方式

你要先决定：

  • 方案 A：单机 Kubernetes
    • 只有一台树莓派，先搭一个单节点集群学习。
  • 方案 B：多机 Kubernetes
    • 比如：
      • 1 台 master/control-plane
      • 1~N 台 worker

下面是 最通用的 kubeadm 方式 ，默认：

  • 系统：Raspberry Pi OS 64-bit / Ubuntu Server 64-bit
  • 架构：arm64
  • 容器运行时：containerd
  • 网络插件：Flannel
  • Kubernetes：v1.29 系列

常见问题排查

问题 1：kubeadm init 失败

看 kubelet 日志：

常见原因：

  • swap 没关
  • cgroup 参数没加
  • containerd 没启动
  • SystemdCgroup = true 没配

问题 2：节点一直 NotReady

通常是网络插件没装好。

查看：

重点看：

  • kube-flannel
  • coredns
